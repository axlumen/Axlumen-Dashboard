# GullDock — Axlumen + Apex 整合计划

## 决策记录

| 决策项 | 选择 |
|--------|------|
| 插件名 | **GullDock** |
| 主题 | Island + Nordic（2个） |
| Banner | 紧凑高度 + 背景图 + 名言轮播 |
| 布局 | 自适应网格（非固定列） |
| 外部服务 | 番茄钟（Sidebar圆环+弹窗）、天气（Sidebar）、TickTick（完整：今日+列表+编辑） |
| 数据存储 | 混合：核心设置 → data.json，卡片/列数据 → Markdown |
| i18n | 不需要（仅中文） |
| 实施方式 | 渐进式，每阶段可测试 |

---

## 阶段 1：基础框架 + 主题系统

**目标：** 替换 axlumen 的 CSS，接入 apex 的主题变量系统（仅 Island + Nordic）

**修改文件：**
- `styles.css` — 从 apex 提取 Island + Nordic 主题变量，替换 axlumen 的 `--ax-*` 为 `--db-*`
- `constants.ts` — 更新设计令牌以匹配 apex 的 `--db-*` 变量
- `types.ts` — 添加 apex 数据模型类型（DashboardData, DashboardColumn, DashboardCard, BannerData, QuickAction）

**新增文件（从 apex 适配）：**
- `src/banner.ts` — 紧凑 Banner（背景图 + 名言轮播，高度 ~60-70px）
- `src/quick-actions.ts` — Glass pill 快捷操作按钮（从 apex 适配）

**验证：** `npm run build` 成功，主题切换正常

---

## 阶段 2：布局重构

**目标：** 用 apex 的布局 shell 替换 axlumen 的布局

**修改文件：**
- `src/view.ts` — 核心重写：
  - Banner: 紧凑高度 + 背景图 + 名言轮播 + 折叠按钮
  - Sidebar: apex 的 hover-to-expand / click-outside-collapse / pin 行为
  - Kanban: 自适应网格布局（CSS Grid，自动列数）
  - 保持 axlumen 的事件系统（vault events, debounce, cooldown）
  - 保持 axlumen 的 LazyRenderer + CompositeDisposable

**新增文件（从 apex 适配）：**
- `src/dnd.ts` — 拖拽排序系统
- `src/recent.ts` — 最近文档组件

**验证：** 侧栏折叠/展开/固定正常，卡片可拖拽排序，Banner 可折叠

---

## 阶段 3：数据模型 + 渲染引擎

**目标：** 建立混合数据模型，引入 apex 的渲染模式

**修改文件：**
- `src/types.ts` — 合并数据模型：
  ```
  DashboardSettings (data.json)
  ├── apex: stylePreset, widgetOrder, pomodoro*, weather*, ticktick*
  ├── axlumen: claudeCodePath, vaultPath, autoRefresh, wikis, projects, automationRules
  
  DashboardData (Markdown file)
  ├── apex: banner, quickActions, columns[]
  └── 新增 sectionType: 'agent-workflows' | 'inbox-router' | 'feeds' | 'vault-health'
  ```
- `src/settings.ts` — 合并设置面板（apex 的视觉设置 + axlumen 的功能设置）

**新增文件（从 apex 适配）：**
- `src/sync.ts` — Markdown 文件数据持久化（用于 DashboardData）
- `src/parser.ts` — Markdown ↔ DashboardData 解析器
- `src/renderer.ts` — 区块渲染引擎（从 apex 适配，添加 axlumen 区块类型）

**验证：** DashboardData 可从 Markdown 文件读写，区块可渲染
