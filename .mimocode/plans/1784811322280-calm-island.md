## 二、布局重构方案

### 2.1 目标架构

从当前的 **CSS Grid 自适应网格** 改为 **Flex Column + Section Row** 模式：

```
当前（Grid）：
┌─────────┬─────────┬─────────┐
│  todos  │ heatmap │ vaultH  │  ← 每个卡片是独立网格单元
├─────────┼─────────┼─────────┤
│  feeds  │projects │ recent  │
└─────────┴─────────┴─────────┘

目标（Section Row）：
┌─────────────────────────────────┐
│ ≡ ▾ 今日待办                     │  ← Section: 完整宽度行
│ ┌──────┐ ┌──────┐ ┌──────┐     │
│ │ Card │ │ Card │ │ Card │ →   │  ← 内部横向滚动
│ └──────┘ └──────┘ └──────┘     │
├─────────────────────────────────┤
│ ≡ ▾ 活跃项目                     │
│ ┌──────┐ ┌──────┐ ┌──────┐     │
│ │ Card │ │ Card │ │ Card │ →   │
│ └──────┘ └──────┘ └──────┘     │
├─────────────────────────────────┤
│ ≡ ▾ 信息流                       │
│ ...                             │
└─────────────────────────────────┘
```

### 2.2 Section 类型定义

| Section | 内部布局 | max-height | 特殊行为 |
|---------|---------|-----------|---------|
| todos | 横向滚动卡片 | 500px | 可拖拽排序任务 |
| projects | 横向滚动卡片 | 无限制 | 封面图片 |
| feeds | 自适应网格 | 800px | GitHub + RSS |
| recent | 列表 | 400px | 点击跳转文件 |
| dashboard | CSS Grid auto-fill | 500px | 番茄钟 + 天气（侧栏已有，可选保留或移除） |

### 2.3 交互系统

每个 Section Row 包含：
1. **Grip Handle**（左侧）— 拖拽排序整个 Section
2. **Toggle Button**（标题前）— 折叠/展开
3. **Title** — Section 名称
4. **Action Buttons**（标题右侧）— 添加卡片、刷新等
5. **Cards Container** — 内部卡片区域
6. **Resize Handle**（底部）— 拖拽调整 Section 高度

## 三、执行步骤

### Phase 1：清理死代码（低风险，立即执行）

**步骤**：

3. 清理 settings.ts 中对应的设置项（TickTick、Manager Audit 等）
4. 清理 main.ts 中对应的命令注册
5. 清理 styles.css 中对应的样式规则
6. 构建验证（`npm run build`）

**删除文件清单**：

### Phase 2：移除不需要的功能模块

**目标**：移除 heatmap、vaultHealth、capabilityManifest

**步骤**：
1. 从 CARD_IDS 数组中移除 `'heatmap'`, `'vaultHealth'`, `'capabilityManifest'`
2. 删除对应的 widget 文件：
   - `src/widgets/heatmap.ts`（334 行）
   - `src/widgets/vaultHealth.ts`（275 行）
   - `src/widgets/capabilityManifest.ts`（184 行）
3. 清理 view.ts 中对应的 `createHeatmapContent()`, `createVaultHealthContent()`, `createCapabilityManifestContent()` 方法
4. 清理 settings.ts 中相关设置
5. 清理 styles.css 中相关样式
6. 构建验证

### Phase 3：CSS 布局重构

**目标**：将 Kanban 从 CSS Grid 改为 Flex Column + Section Row

**修改文件**：`styles.css`

**关键改动**：
1. `.ax-kanban`：`display: grid` → `display: flex; flex-direction: column`
2. `.ax-section-row`：添加 `max-height`、`flex-shrink: 0`、`flex-direction: column`
3. 新增 Section 类型差异化样式（不同类型的 max-height 和内部布局）
4. 新增 `.ax-section-grip`、`.ax-section-toggle`、`.ax-section-resize-handle` 样式
5. 新增拖拽状态样式（`.ax-section-row--dragging`、`.ax-section-row--section-drag-over`）
6. 新增折叠状态样式（`.ax-section-row--collapsed`）
7. 更新响应式断点

### Phase 4：TypeScript 交互系统

**目标**：实现 Section 级别的折叠、拖拽排序、高度调整

**修改文件**：`src/view.ts`

**关键改动**：
1. 重写 `createSectionScaffold()` — 添加 grip、toggle、resize handle
2. 重写 `enhanceCardWithCollapseAndDrag()` — Section 级别交互
3. 新增 `setupSectionDragAndDrop()` — Section 拖拽排序
4. 新增 `setupSectionResize()` — 底部拖拽调整高度
5. 新增 `saveSectionHeight()` / `getSectionHeight()` — 高度持久化
6. 更新 `createCardById()` — 传递 sectionType 参数

### Phase 5：验证

**验证步骤**：
1. `npm run build` — 构建通过
2. 在 Obsidian 中加载插件，检查：
   - 所有保留的面板正常显示
   - Section 可折叠/展开
   - Section 可拖拽排序
   - Section 高度可调整
   - 移动端响应式正常
   - 侧栏模块正常
   - Banner 正常
   - Agent 系统正常
